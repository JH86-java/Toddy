# 🚀 启动问题解决方案

## ❓ 为什么GUI版启动不成功？

可能的原因：
1. **远程连接/SSH** - GUI窗口无法在你的屏幕上显示
2. **后台运行** - 程序已启动，但窗口在你看不到的地方
3. **权限问题** - 需要管理员权限

---

## ✅ 解决方案

### 方案1：使用终端版（推荐⭐）

不需要GUI，直接在终端中使用：

```bash
cd work_tracker

# 添加任务
python terminal_assistant.py add upcoming "完成XX功能"
python terminal_assistant.py add completed "完成了XX功能"
python terminal_assistant.py add important "重要事项"

# 查看所有任务
python terminal_assistant.py list

# 查看指定分类
python terminal_assistant.py list upcoming

# 生成总结
python terminal_assistant.py summary

# 早上提醒
python terminal_assistant.py reminder
```

### 方案2：直接在Windows桌面启动GUI版

**不要通过终端或远程连接启动**，而是：

1. 打开文件资源管理器
2. 导航到项目目录
3. 双击 `run.bat`

这样GUI窗口会直接在你的桌面上显示。

### 方案3：使用诊断工具

双击 `诊断工具.bat` 检查系统环境

## 💡 推荐使用方式

### 日常使用（终端版）

创建快捷命令，方便快速记录：

```bash
# 添加到PATH后，可以在任何地方使用
ta add upcoming "任务内容"
ta list
ta summary
```

### 或者使用工作追踪器

```bash
w.bat add "工作内容"
w.bat report
```

---

## 📋 终端版命令速查

| 命令 | 说明 | 示例 |
|------|------|------|
| `add <分类> <内容>` | 添加任务 | `add upcoming "开会"` |
| `list [分类]` | 列出任务 | `list` 或 `list completed` |
| `delete <分类> <编号>` | 删除任务 | `delete upcoming 1` |
| `summary` | 今日总结 | `summary` |
| `reminder` | 早上提醒 | `reminder` |

**可用分类**：
- `completed` - 已完成
- `in_progress` - 进行中
- `upcoming` - 即将做
- `important` - 重要事项

## 🔧 如果一定要用GUI版

### 步骤1：确保在Windows桌面环境
- 不要通过SSH或远程桌面
- 直接在物理机上操作

### 步骤2：检查Python tkinter
```bash
python -c "import tkinter; print('OK')"
```

### 步骤3：双击启动
```
项目目录\run.bat
```

### 步骤4：检查任务栏
- 查看任务栏是否有Python图标
- 右键点击恢复窗口

## ❓ 常见问题

**Q: 终端版和GUI版数据通用吗？**  
A: 是的，都使用同一个 `.work_data` 目录下的数据文件

**Q: 哪个更好用？**  
A: 终端版更可靠，GUI版更直观。建议先用终端版。

**Q: 如何备份数据？**  
A: 复制整个 `.work_data` 文件夹
