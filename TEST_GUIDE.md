# Toddy 性能优化 - 测试指南

## ✅ 已应用的修复

### 1. 线程安全问题修复（最关键）
- ✅ 后台线程不再直接创建 Tkinter 弹窗
- ✅ 使用队列机制在主线程中处理提醒
- ✅ 每100ms检查一次队列，确保及时响应

### 2. 定时检查优化
- ✅ 收集需要提醒的任务到列表
- ✅ 通过 `root.after()` 调度到主线程执行
- ✅ 避免了跨线程UI操作

## 🔍 如何监控性能

### Windows任务管理器
1. 按 `Ctrl + Shift + Esc` 打开任务管理器
2. 找到 `python.exe` 或 `smart_assistant.py`
3. 观察以下指标:
   - **CPU**: 应该很低（<5%）
   - **内存**: 应该稳定（约50-100MB）
   - **磁盘**: 偶尔读写（保存数据时）

### Python性能监控（可选）
```python
import psutil
import os

process = psutil.Process(os.getpid())
print(f"CPU: {process.cpu_percent(interval=1)}%")
print(f"Memory: {process.memory_info().rss / 1024 / 1024:.2f} MB")
```

### 问题2: 提醒不触发
**检查**:
- 查看控制台输出是否有错误
- 确认任务的 `remind_at` 字段格式正确
- 检查 `_process_reminder_queue_loop` 是否在运行

### 问题3: 界面仍然卡顿
**可能原因**:
- 任务数量过多（100+）
- 需要进一步优化 `update_display` 方法

**临时方案**:
- 减少同时显示的任务数量
- 定期清理已完成的任务
